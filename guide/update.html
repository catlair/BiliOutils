历史配置对比 ​

历史配置对比

v0.23.805

  • 警告 部分云函数平台不再进行压缩包打包。
  • 修复 部分系统下因为日志数据流导致程序直接崩溃。
  • 修复 创建新 cookie 仅仅只有第一个配置生效。
  • 修复 莫名其妙的 24 点。
  • 修复 错误日志并没有输出 stack。
  • 修复 获取稿件：“请求过于频繁，请稍后再试”。
  • 修复 配置中解除投币 5 个限制的配置没作用。
  • 修复 指定 up 主投币但还是会根据顺序给其他 up 主投币。
  • 新增 exchangeBigPoint 大积分兑换。
  • 新增 liveDm 可配置。
  • 新增 dailyBattery 旧版任务下线，新版任务替换（测试中，配置详见历史各版本配置对比）。
  • 新增 mangaTask 夏季活动，没有卵用，白嫖一个漫画装扮而已。
  • 新增 [app.http]配置项，解决部分 api 在不同版本下结果不同的问题。
  • 新增 使用 renovate-bot 自动更新依赖。

修复 漫画阅读额外使用了非即将过期的漫读券。
修复 大积分任务未全部完成的情况下没有日志输出。

警告 删除 serverless 部署工具。

新增 每月会员福利会领取 [漫画商城优惠]。
新增 多账户下风纪委员长期没案件会阻塞后期运行，现可以配置为异步执行。

修复 修复 dailyBattery 任务只部分完成的问题。
修复 粉丝亲密度 intimacy 错误的结束、错误的重试机制。
优化优化 大积分每日 75 调整到 65（官方删除一个任务）。

优化 漫画阅读快速完成。

0.6.221014 ​

修复 【非云函数】，转盘抽奖错误的判断为今日已运行。
新增 为亲密度增加运行后检查运行情况，默认不开启，因为 b 站数据延迟可能会出现检查后从头运行。
新增 【仅云函数】，风纪委员在没有新案件需要休眠时，采用新建 Trigger 等待的方式（通过调用次数换取资源时间）。默认开启 [jury.newTrigger]。
新增 【非云函数】，避免出现某些人所说的莫名奇妙的 cookie 失效（当然只是尝试，本人并遇到过），配置 createCookieDay，运行通过当前 cookie 创建出新的 cookie，（非刷新，cookie 可以共存）。
优化 为亲密度减少某些情况下的 api 调用。
新增 漫画每日首次分享。
新增 【合作视频】精准匹配用户，视频上传者必须是目标用户。这是曾经有过的功能，后来因为逻辑的变动和 api 的更改就废弃了。
优化 获取指定类型稿件的方式由成功获取后进行判断，修改为只查找指定类型。
优化 稿件来源可自定优先顺序和去掉来源。
优化 多账号中错误的配置不再加载时直接跳过，改为运行时跳过。

优化 部分情况的日志输出优化，增加中奖标题提示（例如：xxxxx-【转盘】）（Issue 92）。
